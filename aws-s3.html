1 ภาพรวม (Overview)

Amazon Simple Storage Service (Amazon S3) คือ object storage service ที่ให้บริการเก็บข้อมูลในรูปแบบ object ใน cloud ด้วยความสามารถระดับ enterprise ที่ scale ได้ไม่จำกัด S3 ช่วยให้ผู้ใช้สามารถ:

Store

เก็บทุกไฟล์ — documents, images, videos, datasets, backups ขนาด 0 bytes ถึง 5 TB

Retrieve

ดึงข้อมูลได้จากที่ใดก็ได้ผ่าน internet ด้วย REST API หรือ CLI

Protect

ปกป้องข้อมูลด้วย 11 9's durability (99.999999999%) และ security ระดับสูง

11 9's Durability — ถ้าเก็บ 10,000,000 objects คาดว่าจะเสียหายเฉลี่ย 1 object ต่อ 10,000 ปี มาพร้อม Availability 99.99% (S3 Standard)

S3 ทำอะไรได้บ้าง

  • Provision ไม่จำกัด storage — ไม่ต้องวางแผน capacity ล่วงหน้า
  • Pay เฉพาะสิ่งที่ใช้งานจริง (pay-as-you-go)
  • Manage การเข้าถึง, versioning, lifecycle ได้อย่างละเอียด
  • Analyze ข้อมูลในตัว (S3 Analytics, S3 Select)

2 พื้นฐาน: Buckets และ Objects

2.1 Bucket

Bucket คือ container ระดับบนสุดสำหรับเก็บ objects — เปรียบได้กับ top-level folder

Global Uniqueness

ทุก bucket ต้องมีชื่อไม่ซ้ำกันทั่วโลก (ไม่ใช่แค่ใน account) เพราะใช้ namespace เดียวกันทั้ง AWS

Regional

ถูกเก็บใน specific AWS Region แต่เข้าถึงได้จากทั่วโลกผ่าน global bucket name

Bucket Naming Rules

  • ชื่อ: 3-63 ตัวอักษร
  • ใช้ได้: lowercase letters, numbers, periods (.), hyphens (-)
  • ห้ามเริ่มต้นด้วย period
  • ห้ามจบด้วย period
  • ห้ามใช้ IP format (เช่น 192.168.5.11)
Limit: Default 100 buckets ต่อ account (ขอเพิ่มได้)

2.2 Object

Object คือไฟล์ที่เก็บใน S3 ประกอบด้วย 5 ส่วนหลัก:

Componentคำอธิบาย
Keyชื่อ object รวม path เช่น photos/vacation/beach.jpg
Valueข้อมูล (data content) — ขนาด 0 bytes ถึง 5 TB
Version IDUnique ID สำหรับแต่ละ version ของ object (ถ้าเปิด versioning)
Metadataข้อมูลอธิบาย object — system-defined หรือ user-defined
TagsKey-value pairs สำหรับ labeling (สูงสุด 10 tags ต่อ object)

Object URL Format

https://<bucket-name>.s3.<region>.amazonaws.com/<key>
# หรือ
https://s3.<region>.amazonaws.com/<bucket-name>/<key>

2.3 S3 Prefixes และ Folder Structure

S3 ไม่มี directory แท้ๆ แต่ใช้ key prefixes จำลองโครงสร้าง folder ได้:

photos/
  vacation/
    beach.jpg        → key: "photos/vacation/beach.jpg"     (prefix: "photos/vacation/")
  profile/
    avatar.png       → key: "photos/profile/avatar.png"   (prefix: "photos/profile/")

3 Storage Classes

S3 มี 8 storage classes เพื่อรองรับ use case และ cost ที่แตกต่างกัน ตามความถี่เข้าถึงข้อมูล

Storage Class ความถี่เข้าถึง Retrieval Time Min Duration Use Case
S3 Standardบ่อยInstant (<ms)NoneActive data, web hosting
S3 Intelligent-Tieringเปลี่ยนแปลงInstantNoneUnknown patterns, data lakes
S3 Standard-IAน้อยInstant (<ms)30 daysDR, long-term storage
S3 Glacier InstantหายากมากInstant (<1 sec)90 daysArchival ที่ต้อง access ทันที
S3 Glacier Flexibleหายากมาก1 min – 12 hr90 daysTape replacement
S3 Glacier Deep Archiveหายากที่สุด12–48 hr180 daysCompliance, 10yr+ retention
S3 One Zone-IAน้อยInstant (<ms)30 daysReproducible data (ไม่ resilient ต่อ AZ failure)
S3 Express One Zoneบ่อยมากSingle-digit msNoneLatency-sensitive workloads

3.1 S3 Intelligent-Tiering — Auto-Tiers

ย้าย object ระหว่าง tiers อัตโนมัติตาม usage pattern โดยไม่มี retrieval charges:

Tiers

• Frequent Access — สำหรับ object ที่ใช้บ่อย
• Infrequent Access — ไม่เข้าถึง 30+ วัน
• Archive Instant — ไม่เข้าถึง 90+ วัน
• Deep Archive — ไม่เข้าถึง 180+ วัน

Cost

ค่า storage เหมือน Standard บวกค่า monitoring $0.0025 per 1,000 objects ต่อเดือน เหมาะสำหรับ access pattern ที่ไม่แน่นอน

OptionเวลาUse Case

Standard3–12 ชั่วโมงStandard retrieval
Bulk5–12 ชั่วโมงLarge amounts, free

4 S3 Features

4.1 S3 Versioning

เก็บรักษาทุก version เมื่อ overwrite หรือ delete — ดึง version เก่ากลับมาได้เสมอ

  • เปิด/ปิดได้ระดับ bucket
  • เก็บทุก version รวมถึง delete markers
  • สามารถ suspend versioning ได้
  • มี Lifecycle rules สำหรับจัดการ noncurrent versions
Delete behavior: Delete object โดยไม่ระบุ version → สร้าง delete marker ไม่ใช่ลบจริง · Delete version เฉพาะ → ลบจริง · ลบ delete marker → restore object ได้

4.2 S3 Replication

CRR — Cross-Region

Replicate ไปยัง bucket อื่นคนละ Region เหมาะสำหรับ DR และลด latency

SRR — Same-Region

Replicate ใน Region เดียวกัน เหมาะสำหรับ backup และ compliance

S3 RTC (Replication Time Control) — Replicate ภายใน 15 นาที guaranteed สำหรับ workload ที่ต้องการ RTO ต่ำ

4.3 S3 Multipart Upload

แบ่ง upload เป็นหลายส่วนสำหรับไฟล์ขนาดใหญ่ (5 MB – 5 TB) แนะนำสำหรับไฟล์ > 100 MB:

1. Initiate Multipart Upload  → ได้ Upload ID
2. Upload Parts                →  ส่ง Part Number + ETag กลับมา
3. Complete Multipart Upload   →  S3 ประกอบ parts เป็น object
Benefits: Resume ได้ถ้าล้มเหลว (retry เฉพาะ part ที่ล้มเหลว) · เร็วขึ้นเพราะ upload parallel · รองรับไฟล์ 5 TB

4.4 S3 Select

Query object โดยตรงด้วย SQL-like syntax — ดึงเฉพาะ subset ที่ต้องการ:

Formatรองรับ
InputCSV, JSON, Parquet, GZIP, BZIP2
OutputCSV, JSON
ลด data transfer อย่างมาก — บางครั้งถึง 99% เมื่อต้องการ subset ของ large dataset

5 Security

5.1 S3 Block Public Access

ตั้งค่าระดับ account หรือ bucket เพื่อป้องกัน accidental public exposure — Default: enabled สำหรับ bucket ใหม่ทุก bucket

Governance Mode

IAM permission พิเศษเท่านั้นที่ bypass ได้ — เหมาะสำหรับควบคุม organizational-wide

Compliance Mode

ไม่มีใคร bypass ได้เด็ดขาด (รวมถึง root) จนกว่าจะหมด retention period

5.2 Access Control

MethodคำอธิบายRecommendation
IAM Policiesควบคุม access ผ่าน IAM users/roles✅ แนะนำใช้เมื่อเป็นไปได้
Bucket PoliciesJSON policy ที่ attach กับ bucket✅ ดีสำหรับ cross-account
ACLsLegacy permission model⚠️ ใช้เมื่อจำเป็นต้อง grant แยกราย object
Access PointsNamed endpoints สำหรับแต่ละ use case✅ ดีสำหรับ complex access patterns

5.3 Encryption

TypeKey ManagementNotes
SSE-S3S3-managed (AES-256)✅ เรียบง่าย, S3 จัดการทุกอย่าง
SSE-KMSAWS KMS keysAudit ได้ละเอียด, ควบคุม key rotation
SSE-CCustomer-provided keysS3 ไม่เก็บ key, ลูกค้าต้องจัดการเอง
CSEClient-side (ก่อน upload)ใช้ AWS Encryption SDK หรือ library อื่น

5.4 S3 Object Lock (WORM)

ป้องกัน object ถูก overwrite หรือ delete ตามกฎหมาย:

Modeป้องกันBypass
Governanceลบ/overwrite ไม่ได้โดยไม่มี permissionIAM permission พิเศษ bypass ได้
Complianceลบ/overwrite ไม่ได้เด็ดขาดไม่มีใคร bypass ได้ จนหมด retention
Legal Hold: ไม่มีวันหมด — ปลดได้เมื่อต้องการ (ต้องมี s3:PutObjectLegalHold permission)

6 Data Protection และ Backup

6.1 S3 Lifecycle Policies

Automate object transitions และ expirations อัตโนมัติ:

Actionคำอธิบาย
Transition to Standard-IAย้ายหลัง 30 วัน (default min)
Transition to Glacierย้ายหลัง 90 วัน (default min)
Transition to Deep Archiveย้ายหลัง 180 วัน (default min)
Object Expirationลบ object หลัง X วัน
Abort Incomplete Multipart Uploadลบ parts ที่ยังไม่ complete หลัง X วัน
Noncurrent Versionจัดการ previous versions หลัง X วัน

6.2 Backup Options

S3 Replication

Sync ไป bucket อื่น (same หรือ cross-region) อัตโนมัติ

AWS Backup

Centralized backup รวม S3 กับ services อื่นๆ

Object Lock

Immutable storage สำหรับ compliance และ litigation

7 Networking และ Data Access

TypeDescriptionค่าใช้จ่าย

Interface Endpointใช้ ENI + private IP (PrivateLink)มีค่าใช้จ่าย

7.2 S3 Event Notifications

Trigger actions เมื่อเกิด events บน bucket:

Event Types

s3:ObjectCreated:* · s3:ObjectRemoved:* · s3:ObjectRestore:* · s3:LifecycleExpiration:*

Destinations

SNS Topic · SQS Queue · Lambda Function · EventBridge

7.3 S3 Transfer Acceleration

ใช้ CloudFront edge locations รอบโลกเพื่อ accelerate uploads/downloads:

https://mybucket.s3-accelerate.amazonaws.com
เร็วขึ้นโดยเฉพาะ distance ที่ไกล — routing ผ่าน AWS backbone network แทน public internet

8 S3 Batch Operations

จัดการ object จำนวนมากใน operation เดียว:

Use Cases

• Copy/replicate existing objects
• Replace ACLs หรือ tags
• Invoke Lambda บน objects
• Encrypt/decrypt objects
• Create cross-region replicas

How It Works

1. สร้าง Job ระบุ objects ผ่าน Inventory manifest หรือ CSV
2. S3 Batch Operations ประมวลผลทุก object
3. ได้ Completion report พร้อมรายละเอียด failure

9 Performance และ Optimization

9.1 S3 Request Rate

3,500 PUT/COPY/POST/DELETE และ 5,500 GET/HEAD ต่อวินาที ต่อ prefix — ไม่มี limits ต่อ bucket

9.2 Performance Best Practices

  • Randomize prefix — กระจาย requests ข้าม prefixes ด้วย key naming ที่ random (เช่น เพิ่ม hash หรือ UUID)
  • CloudFront — cache GET requests ลด request load บน S3
  • Multipart Upload — แบ่ง upload หลายส่วน parallel
  • S3 Transfer Acceleration — ใช้ edge locations สำหรับ uploads จาก remote locations
  • S3 Select — query เฉพาะ data ที่ต้องการแทนดึงทั้งไฟล์

9.3 Multipart Upload Optimization

Part SizeUse Case
8–64 MBStandard large files
512 MBVery large files (maximum parallelism)

      "TargetBucket": "my-log-bucket",
      "TargetPrefix": "logs/"
    }
  }'

11 Static Website Hosting

S3 สามารถ host static website ได้ทันที:

aws s3 website s3://my-bucket/ \
  --index-document index.html \
  --error-document error.html
หมายเหตุ: Website endpoints ไม่ใช่ HTTPS — ถ้าต้องการ HTTPS ใช้ CloudFront + OAI หรือ S3 + CloudFront custom domain

Website endpoint format:

http://my-bucket.s3-website.<region>.amazonaws.com

ขั้นตอน:

  1. เปิด Static website hosting ใน bucket properties
  2. เปิด Block public access (หรือใช้ CloudFront)
  3. เพิ่ม bucket policy ให้ public read สำหรับ objects
